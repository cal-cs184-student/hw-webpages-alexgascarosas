Overview

In this homework, we implemented several geometric modeling techniques related to Bezier curves, Bezier surfaces, and triangle meshes. Overall, the assignment focused on understanding how smooth curves and surfaces can be represented mathematically and how triangle meshes can be manipulated using the half-edge data structure.

In the first section, we implemented Bezier curve evaluation using de Casteljau’s algorithm. The idea behind the algorithm is to repeatedly linearly interpolate between control points using a parameter t. Each step creates a smaller set of intermediate points until only one point remains, which lies on the Bezier curve. We implemented the function that performs one step of this interpolation and then used it recursively to evaluate the curve. Next, we extended this idea to Bezier surfaces. Instead of evaluating a single curve, the surface evaluation works by applying the same 1D algorithm in two directions. First, each row of control points is treated as a Bezier curve and evaluated using parameter u. This produces a set of points that form another curve, which is then evaluated using parameter v to get the final point on the surface.

In the second part of the homework, we worked with triangle meshes using the half-edge data structure. We implemented area-weighted vertex normals, which are used for smooth shading. To compute this, we iterated over all faces connected to a vertex, computed each face’s normal using a cross product, weighted it by the face area, and summed them together before normalizing the result. After that, we implemented two mesh operations: edge flips and edge splits. Edge flipping changes the diagonal shared by two triangles without adding or removing elements, while edge splitting inserts a new vertex at the midpoint of an edge and creates additional edges and faces. These parts required carefully updating many pointers in the half-edge structure to make sure all vertices, edges, faces, and halfedges still pointed to the correct elements. Finally, we implemented loop subdivision to upsample the mesh. This algorithm subdivides each triangle into four smaller triangles and updates vertex positions using weighted averages of neighboring vertices. I first computed the new positions for vertices and edge midpoints, then performed edge splits and flips to subdivide the mesh, and finally updated the vertex positions.

One thing we found interesting about homework 2 is how much detail is involved in working with meshes. The math concepts like interpolation and subdivision are fairly straightforward, but implementing them correctly with the half-edge data structure required careful debugging and attention to how everything is connected.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm takes a list of control points and a scalar parameter t in [0, 1], and recursively produces a new list of points where each point lies on the line segment between two consecutive points from the previous level. The precise position is determined by t: when t = 0 the point coincides with points[i], and when t = 1 it coincides with points[i+1]. In my implementation, I loop from index 0 to (size - 2), linearly interpolating each consecutive pair (points[i], points[i+1]) to build the next level, then repeat this process on successive levels until a single point remains.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau's algorithm extends to Bezier surfaces by applying it separably along each parameter axis. Given an n x n grid of control points and two parameters u and v, I first run the full 1D algorithm along each row at parameter u, reducing each row of n points down to a single point. This produces n new points, one per row, which together define a new Bezier curve running in the other direction. I then run the full 1D algorithm on those n points at parameter v to obtain the final surface point P(u, v). In my implementation, evaluate1D repeatedly calls evaluateStep until only one point remains, and evaluate iterates over each row of controlPoints, calling evaluate1D with u to collect the n intermediate points, then calls evaluate1D again on those points with v to produce the final result.

Part 5: Edge split

For edge split, given an edge (b,c) shared by triangles (a,b,c) and (c,b,d), I insert a new vertex m at the midpoint of (b,c) and create 4 new triangles: (a,c,m), (a,b,m), (b,d,m), (c,d,m). I allocate 8 new halfedges, 4 new edges, 4 new faces, and 1 new vertex, then set all neighbor pointers via setNeighbors for every halfedge in the region, including the 4 border halfedges whose next and face pointers change. After wiring everything up, I delete the old edge, its two halfedges, and the two old faces, rather than reassigning them. For an implementation trick, I listed out every relevant mesh element upfront and named them directly after the diagram shown in lecture, which made the setNeighbors calls straightforward to write and verify against the picture. I also chose to delete old elements instead of reassigning pointers, which added a few extra lines but made the logic easier to follow. Debugging was tedious since the counter clockwise winding order tripped me up a lot since my brain naturally goes clockwise, so I kept getting next pointers backwards. Most of the debugging was manually checking each setNeighbors call one by one against the diagram. Drawing the before and after diagrams with labeled halfedges was essential.
